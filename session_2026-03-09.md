# T-Ragx Session Notes — 2026-03-09

## What We Did

### 1. Repo Audit
Explored the full T-Ragx codebase and mapped out the translation pipeline:
- **`TRagx`** (`src/t_ragx/main.py`) — orchestrator: retrieves RAG context then calls LLM
- **`ElasticInputProcessor`** (`src/t_ragx/processors/`) — fetches glossary (S3 Parquet) and translation memory (Elasticsearch)
- **`BaseModel`** (`src/t_ragx/models/`) — builds prompt, tokenizes, generates, decodes
- Supported backends: HuggingFace, llama-cpp-python, Ollama, OpenAI-compatible API

### 2. Glossary Search Deep-Dive
Traced `batch_search_glossary()` step by step:
1. `batch_search_glossary()` → loops and calls `search_glossary()` per item
2. `search_glossary()` → Unicode normalizes input, checks task glossary first, fills remainder from general glossary
3. `search_general_glossary()` → lazy-loads Parquet from S3 (ETag-cached to `/tmp/t_ragx/`), passes dict to `get_glossary()`
4. `get_glossary()` → O(n) substring scan over glossary dict; whole-word matching for English source; deduplication of sub-terms
5. Returns `{source_term: [translation1, translation2, ...]}` injected into LLM prompt
